# Reverse Proxy & TLS

MimDB listens on HTTP and relies on a reverse proxy for TLS termination. Nginx
and Caddy are both fully supported. Cloudflare Tunnel is recommended for
zero-infrastructure TLS without managing certificates.

## Nginx

### Install and get a certificate

```bash
sudo apt install nginx certbot python3-certbot-nginx
sudo certbot --nginx -d api.yourdomain.com
```

### Configuration

Create `/etc/nginx/sites-available/mimdb`:

```nginx
upstream mimdb {
    server 127.0.0.1:9000;
    keepalive 32;
}

# Redirect HTTP to HTTPS
server {
    listen 80;
    listen [::]:80;
    server_name api.yourdomain.com;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    listen [::]:443 ssl http2;
    server_name api.yourdomain.com;

    ssl_certificate /etc/letsencrypt/live/api.yourdomain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/api.yourdomain.com/privkey.pem;
    ssl_session_timeout 1d;
    ssl_session_cache shared:MimDB:10m;
    ssl_session_tickets off;
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers ECDHE-ECDSA-AES128-GCM-SHA256:ECDHE-RSA-AES128-GCM-SHA256:ECDHE-ECDSA-AES256-GCM-SHA384:ECDHE-RSA-AES256-GCM-SHA384;
    ssl_prefer_server_ciphers off;

    add_header Strict-Transport-Security "max-age=63072000; includeSubDomains" always;

    # Allow large bodies for storage uploads
    client_max_body_size 100m;

    # Timeouts
    proxy_connect_timeout 10s;
    proxy_send_timeout 60s;
    proxy_read_timeout 60s;

    # Standard proxy headers
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;

    # HTTP/1.1 keep-alive to upstream
    proxy_http_version 1.1;
    proxy_set_header Connection "";

    # Health check (suppress access logs)
    location = /health {
        proxy_pass http://mimdb;
        access_log off;
    }

    # WebSocket upgrade for realtime subscriptions
    location ~ ^/v1/realtime/[^/]+$ {
        proxy_pass http://mimdb;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_read_timeout 3600s;
        proxy_send_timeout 3600s;
    }

    # Storage uploads - extended timeouts for large files
    location ~ ^/v1/storage/ {
        proxy_pass http://mimdb;
        proxy_read_timeout 300s;
        proxy_send_timeout 300s;
        client_max_body_size 500m;
    }

    # Everything else
    location / {
        proxy_pass http://mimdb;
    }
}
```

Enable and reload:

```bash
sudo ln -s /etc/nginx/sites-available/mimdb /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

<Warning>
The realtime location block requires both `Upgrade` and `Connection "upgrade"`
headers. Without these, WebSocket handshakes will fail with `101 Switching
Protocols` never being sent. Do not remove them or merge that location into the
catch-all block.
</Warning>

### Nginx tuning

Add to the `http` block in `/etc/nginx/nginx.conf`:

```nginx
worker_connections 2048;
multi_accept on;

proxy_buffer_size 8k;
proxy_buffers 8 16k;
proxy_busy_buffers_size 32k;

# Gzip JSON responses
gzip on;
gzip_types application/json text/plain;
gzip_min_length 256;
gzip_vary on;
```

## Caddy

Caddy handles TLS certificate issuance and renewal automatically via Let's
Encrypt.

Create `/etc/caddy/Caddyfile`:

```caddyfile
api.yourdomain.com {
    # WebSocket support for realtime
    @realtime path_regexp realtime ^/v1/realtime/[^/]+$
    handle @realtime {
        reverse_proxy 127.0.0.1:9000
    }

    # Storage uploads - extended timeouts
    @storage path /v1/storage/*
    handle @storage {
        reverse_proxy 127.0.0.1:9000 {
            transport http {
                read_timeout 300s
                write_timeout 300s
            }
        }
    }

    # Everything else
    handle {
        reverse_proxy 127.0.0.1:9000
    }

    request_body {
        max_size 100MB
    }

    header {
        Strict-Transport-Security "max-age=63072000; includeSubDomains"
        -Server
    }

    log {
        output file /var/log/caddy/mimdb-access.log
    }
}
```

Caddy automatically handles WebSocket upgrades for the realtime route - the
`Upgrade` and `Connection` headers are proxied transparently.

## Cloudflare

If your domain is proxied through Cloudflare, configure the following:

### DNS and SSL mode

Create an `A` record pointing to your server's IP with the orange cloud
(proxied) toggle enabled. Under **SSL/TLS**, set the encryption mode to
**Full (strict)** - this requires a valid certificate on your origin server.
Never use "Flexible" mode as it sends credentials over unencrypted HTTP between
Cloudflare and your origin.

### Real client IPs

Cloudflare replaces the client IP with its edge IP. MimDB needs the real IP
for rate limiting and audit logging. Set the trusted IP header in your MimDB
config:

```toml
[rate_limit]
trusted_ip_header = "CF-Connecting-IP"
```

Add this to the `http` block in Nginx to restore the real IP in Nginx logs:

```nginx
set_real_ip_from 173.245.48.0/20;
set_real_ip_from 103.21.244.0/22;
set_real_ip_from 103.22.200.0/22;
# ... (full list at https://cloudflare.com/ips-v4)
real_ip_header CF-Connecting-IP;
```

### Cloudflare Tunnel (no Nginx required)

Cloudflare Tunnel routes traffic directly from Cloudflare's edge to MimDB on
your server without opening any inbound ports:

```bash
# Install cloudflared
curl -L https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-amd64 \
  -o /usr/local/bin/cloudflared
chmod +x /usr/local/bin/cloudflared

# Authenticate and create tunnel
cloudflared tunnel login
cloudflared tunnel create mimdb
cloudflared tunnel route dns mimdb api.yourdomain.com
```

Create `/etc/cloudflared/config.yml`:

```yaml
tunnel: <your-tunnel-id>
credentials-file: /root/.cloudflared/<your-tunnel-id>.json

ingress:
  - hostname: api.yourdomain.com
    service: http://127.0.0.1:9000
  - service: http_status:404
```

Run as a system service:

```bash
sudo cloudflared service install
sudo systemctl start cloudflared
```

With a tunnel active, you can restrict your firewall to SSH only - no port 80
or 443 needs to be open.

### Network settings

Enable **WebSockets** in the Cloudflare dashboard under **Network** - this is
required for Realtime subscriptions. Disable **Rocket Loader** and **Auto
Minify** to avoid interference with API responses.

## CORS

CORS preflight responses are generated by MimDB itself, not the reverse proxy.
The preflight cache duration is controlled by `[server].cors_max_age`
(default: `86400` seconds / 24 hours). Do not add CORS headers in Nginx or
Caddy - doubling them causes browser errors.

Allowed origins for WebSocket connections are configured separately via
`[realtime].allowed_origins`. HTTP CORS and WebSocket origins are independent
settings.
